# B题《多工序协同作业问题》建模思路阐述文档

> 本文档依据《2026五一杯B题建模思路.pdf》整理，目标是形成一份便于项目开发、模型说明和代码实现使用的 Markdown 版思路文档。本文档只对原建模思路进行结构化整理和表达，不改变其基本题意与建模方向。

---

## 1. 问题总体定位

B题属于典型的 **工序调度问题（Job Shop Scheduling）**。

其核心目标是在设备资源有限、工序存在先后关系、设备跨车间转运存在时间成本的条件下，为多个车间的多道工序合理分配设备，使所有任务的总完成时间最短，即最小化 **Makespan**。

从建模角度看，本题不是预测问题，也不是评价问题，而是一个以 **时间最短化** 为目标的资源约束调度优化问题。

### 2.2 C车间特殊循环说明

C车间中，C3、C4、C5 三道工序需要重复执行 3 遍。

等价处理方式为：

```text
C1 → C2 → C3_1 → C4_1 → C5_1 → C3_2 → C4_2 → C5_2 → C3_3 → C4_3 → C5_3
```

其中，C3、C4、C5 各自被拆解为 3 个独立子工序，共形成 9 道循环子工序。

该处理方式的意义是：重复执行不能简单理解为“时间乘以3”，而应作为独立工序链处理，因为每一轮都会占用设备、影响设备可用时间和后续调度。

## 5. 核心计算规则

### 5.1 工序持续时间计算

对于单设备工序，设：

- 工程量为 \(Q\)，单位为 m³；
- 分配设备台数为 \(n\)；
- 单台设备效率为 \(v\)，单位为 m³/h。

则该类设备完成该工序的作业时间为：

```math
T = \left\lceil \frac{Q}{n \cdot v} \times 3600 \right\rceil
```

其中，\(T\) 的单位为秒。

### 5.2 双设备并行工序完成时间

如果某工序需要两类设备 \(X\) 和 \(Y\) 同时参与，且两类设备都需要独立完成全部工程量，则：

```math
T_X = \left\lceil \frac{Q}{n_X \cdot v_X} \times 3600 \right\rceil
```

```math
T_Y = \left\lceil \frac{Q}{n_Y \cdot v_Y} \times 3600 \right\rceil
```

工序完成时间为：

```math
T_{op}=\max(T_X,T_Y)
```

该规则的核心含义是：两类设备并不是分摊工程量，而是各自完成完整工程量；只有两类设备均完成后，该工序才完成。

### 5.3 设备转运时间计算

设设备从位置 \(P_1\) 转运到位置 \(P_2\)，距离为 \(d\)，移动速度为 \(v_m\)，则转运时间为：

```math
T_{trans}=\left\lceil \frac{d}{v_m} \right\rceil
```

本题中，所有设备移动速度均为 2 m/s，因此：

```math
T_{trans}=\left\lceil \frac{d}{2} \right\rceil
```

关键规则：

1. 同一车间内不同工序之间转运时间忽略，记为 0。
2. 跨车间转运时间不可忽略，需要计入设备到达时间。
3. 设备从班组出发到第一个作业车间，也需要计算初始转运时间。

### 5.4 工序先后约束

每个车间内工序必须严格按编号顺序执行。

若工序 \(i\) 是工序 \(i+1\) 的前序工序，则：

```math
S_{i+1} \ge F_i
```

其中：

- \(S_i\)：工序 \(i\) 的开始时间；
- \(F_i\)：工序 \(i\) 的结束时间。

对于 C 车间循环工序，需要满足：

```math
S_{C3}^{(k+1)} \ge F_{C5}^{(k)}, \quad k=1,2
```

即后一轮 C3 必须在上一轮 C5 完成后才能开始。

## 6. 问题1建模思路：班组1独立完成 A 车间任务

### 6.1 问题特征

问题1只涉及：

- A车间；
- 班组1设备；
- A1、A2、A3 三道**工序级**串行（A1 全部完成后才能开始 A2，A2 全部完成后才能开始 A3）；
- 初始设备从班组1出发到 A 车间。

**层次区分（易错点）：**

- **工序之间**：A1→A2→A3 串行，不存在跨工序并行。
- **工序内部**：凡班组1拥有多台**同一设备类型**，在该工序中可**同时开工、并联作业**，等效于把该类型承担的总工程量按台数分摊，单台持续作业时间为「全工程量 /（投入台数 × 单台效率）」向上取整到秒；**表1** 中应对**每一台实际参与作业的设备**各写一行，同类型多台若并行，则起止时刻相同、持续时间为该单台分摊后的作业时长（与题目示例表1一致）。
- **双设备工序**（如 A1）：两类设备各自并行子集，该工序阶段时长为 \(\max(T_{\text{类型1}}, T_{\text{类型2}})\)；两类设备**各自**完成该工序对应的全部工程量（赛题假设(2)），通过增加台数使两类耗时尽量**平衡**，可缩短该阶段瓶颈。

问题1的本质是：在单车间、单班组、**工序链串行**的前提下，对每一道工序决策「各设备类型投入台数」（不超过班组拥有量），必要时在枚举的 \((n_{灌},n_{臂})\)、\((n_{洗},\ldots)\) 上最小化 A3 完工时刻（Makespan）；**不是**「整条产线只用一台设备串行」。

### 6.2 A车间工序计算逻辑

#### A1：缺陷填补

设备需求：精密灌装机 + 自动化输送臂。

工程量：300 m³。

班组1可用设备：

- 精密灌装机 5 台；
- 自动化输送臂 4 台。

设分配：

- \(n_{灌}\) 台精密灌装机；
- \(n_{臂}\) 台自动化输送臂。

则：

```math
T_{灌}=\left\lceil \frac{300}{n_{灌}\cdot 200}\times3600 \right\rceil
```

```math
T_{臂}=\left\lceil \frac{300}{n_{臂}\cdot 250}\times3600 \right\rceil
```

```math
T_{A1}=\max(T_{灌},T_{臂})
```

由于 A1 是第一道工序，设备从班组1到 A 车间距离为 400 m，移动速度为 2 m/s：

```math
T_{trans,班组1\to A}=\left\lceil \frac{400}{2} \right\rceil = 200s
```

因此 A1 的开始时间不早于 200s。

#### A2：表面整平

设备需求：高速抛光机 + 工业清洗机。

工程量：500 m³。

班组1可用设备：

- 高速抛光机 1 台；
- 工业清洗机 5 台。

A2 必须在 A1 完成后才能开始：

```math
S_{A2}\ge F_{A1}
```

同车间内设备转运时间为 0。

高速抛光机固定为 1 台，其时间为：

```math
T_{抛}=\left\lceil \frac{500}{1\cdot100}\times3600 \right\rceil=18000s
```

工业清洗机时间为：

```math
T_{洗}=\left\lceil \frac{500}{n_{洗}\cdot250}\times3600 \right\rceil
```

A2 完成时间为：

```math
T_{A2}=\max(T_{抛},T_{洗})
```

#### A3：强度检测

设备需求：自动传感多功能机。

工程量：500 m³。

班组1可用自动传感多功能机 1 台，效率为 100 m³/h。

```math
T_{A3}=\left\lceil \frac{500}{1\cdot100}\times3600 \right\rceil=18000s
```

A3 必须在 A2 完成后开始：

```math
S_{A3}\ge F_{A2}
```

### 6.3 问题1决策变量

对需要多台设备参与的工序 \(i\)，设分配给该工序设备类型 \(k\) 的台数为：

```math
n_{i,k}\in\mathbb{Z}^{+}
```

### 6.4 问题1约束条件

| 约束 | 表达 | 含义 |
|---|---|---|
| 设备总量约束 | \(n_{i,k}\le N_k^{(1)}\) | 每道工序中使用的同类设备台数不超过班组1拥有数量 |
| 工序先后约束 | \(S_{A2}\ge F_{A1},\ S_{A3}\ge F_{A2}\) | A车间工序串行执行 |
| 初始转运约束 | \(S_{A1}\ge 200\) | 设备从班组1到 A 车间需要 200s |

### 6.5 问题1目标函数

```math
\min F_{A3}
```

即最小化 A 车间最后一道工序 A3 的完成时间。

### 6.6 问题1求解策略

由于 A 车间工序串行，且可选设备台数有限，可以采用枚举法。

具体步骤：

1. 对 A1，枚举 \(n_{灌}=1,2,3,4,5\)，\(n_{臂}=1,2,3,4\)，选择使 \(\max(T_{灌},T_{臂})\) 最小的组合；在表1 中为所选的每台灌装机、每台输送臂各输出一行，**同一子类内**起止时间一致。
2. 对 A2，枚举 \(n_{洗}=1,2,3,4,5\)，高速抛光机台数 \(n_{抛}=1\)（班组仅 1 台），选择使 \(\max(T_{抛},T_{洗})\) 最小的组合；抛光与清洗可**同时开工**，工序完成时刻为 \(\max\)（两类设备各自的最后完工时刻），与「不考虑两类设备之间先后顺序与等待」一致。
3. A3 固定使用 1 台自动传感多功能机（班组仅 1 台）。
4. 将初始转运时间（各机到位后取可开工的公共下界）与三道工序阶段时长按链串行累加，得到总最短时长；工程实现上可在 `solve_problem1` 中对单设备类、双设备类分别枚举投入台数并更新每台设备的可用时刻。

## 7. 问题2建模思路：班组1完成五个车间全部任务

### 7.1 问题新增复杂性

相较于问题1，问题2增加了以下复杂度：

| 新增复杂性 | 说明 |
|---|---|
| 多车间调度 | A、B、C、D、E 五个车间都需要完成 |
| 跨车间转运 | 设备完成一个车间工序后转移到另一车间，会产生转运时间 |
| 设备竞争 | 班组1设备数量有限，不同车间可能同时争用同类设备 |
| 工序依赖 | 同一车间内必须串行，不同车间之间在资源允许时可并行 |
| C车间循环 | C3→C4→C5 需要循环执行3遍 |

### 7.2 集合定义

| 符号 | 含义 |
|---|---|
| \(W=\{A,B,C,D,E\}\) | 车间集合 |
| \(J\) | 工序集合，包括 A1-A3、B1-B4、C1-C5循环展开、D1-D6、E1-E3 |
| \(K\) | 设备类型集合 |
| \(N_k\) | 班组1中设备类型 \(k\) 的台数 |

### 7.3 参数定义

| 参数 | 含义 |
|---|---|
| \(Q(j)\) | 工序 \(j\) 的工程量 |
| \(eff(j,k)\) | 工序 \(j\) 中设备类型 \(k\) 的单台效率 |
| \(dist(w_1,w_2)\) | 车间 \(w_1\) 到 \(w_2\) 的距离 |
| \(v_m\) | 设备移动速度，取 2 m/s |

### 7.4 决策变量

| 变量 | 含义 |
|---|---|
| \(S(j)\) | 工序 \(j\) 的开始时间 |
| \(F(j)\) | 工序 \(j\) 的结束时间 |
| \(n(j,k)\) | 工序 \(j\) 分配的 \(k\) 类设备台数 |
| \(x(e,j)\) | 设备 \(e\) 是否参与工序 \(j\) 的 0-1 变量 |

### 7.5 资源约束

任意时刻，同类设备的占用数量不能超过班组1拥有的总台数。

```math
\sum_{j\in active(t)} n(j,k) \le N_k,\quad \forall k
```

### 7.6 工序先后约束

若工序 \(j\) 是工序 \(j'\) 在同一车间中的前序工序，则：

```math
S(j')\ge F(j)
```

### 7.7 跨车间转运约束

设备从车间 \(w_1\) 完成任务后，转移到车间 \(w_2\) 执行下一任务时，应满足：

```math
S(j')\ge F(j)+\left\lceil \frac{dist(w_1,w_2)}{2}\right\rceil
```

设备从班组1出发到第一个作业车间时：

```math
S(j_{first,w})\ge \left\lceil \frac{dist(班组1,w)}{2}\right\rceil
```

### 7.8 目标函数

```math
\min Makespan=\max_{j\in J}F(j)
```

### 7.9 求解策略

由于该问题属于车间调度的一般形式，具有较高复杂度，可采用 **贪心 + 模拟调度（列表调度算法）**。

基本流程：

1. 预计算每道工序的最短持续时间。
2. 采用关键路径法（CPM）计算每道工序从当前节点到最终完成节点的最长时间链。
3. 优先调度关键路径长度更长的工序。
4. 维护每台设备的最早可用时间和当前位置。
5. 按优先级逐步分配设备、确定开始时间、更新设备状态。
6. 对初始调度结果进行局部优化，例如邻域交换或工序重排。

## 8. 问题3建模思路：班组1和班组2协同完成五个车间任务

### 8.1 问题新增复杂性

相较于问题2，问题3新增：

| 新增复杂性 | 说明 |
|---|---|
| 资源池扩大 | 班组1和班组2设备均可参与调度 |
| 初始位置不同 | 班组2到各车间的初始转运时间与班组1不同 |
| 班组协作 | 两个班组设备可以并行调度 |
| 分配决策更复杂 | 需要决定工序由哪个班组或两个班组联合完成 |

建模思路文档中特别指出：题目未明确限制两个班组设备是否能混用完成同一工序，因此通常可假设两个班组设备能够在同一工序中联合使用，只要设备能及时到达。若实际题目限制不允许混用，则应按限制处理。

### 8.2 设备池扩展

定义设备：

```math
E(g,k,i)
```

表示班组 \(g\) 中第 \(i\) 台类型为 \(k\) 的设备，其中 \(g=1,2\)。

总可用设备数量为：

```math
N_k^{total}=N_k^{(1)}+N_k^{(2)}
```

### 8.3 转运时间扩展

对于设备 \(E(g,k,i)\)，从位置 \(L_1\) 转移到位置 \(L_2\) 的转运时间为：

```math
T_{trans}(L_1,L_2)=\left\lceil \frac{dist(L_1,L_2)}{2}\right\rceil
```

区别在于，班组1和班组2的初始位置不同，因此初始出发时间需要分别计算。

### 8.4 设备分配决策变量

设：

```math
n_{j,k}^{(g)}
```

表示工序 \(j\) 分配班组 \(g\) 的 \(k\) 类设备台数。

### 8.5 目标函数

问题3目标函数仍为：

```math
\min Makespan=\max_{j\in J}F(j)
```

### 8.6 求解策略

建模思路文档给出三种可选求解方法。

#### 方法一：两阶段策略

第一阶段：确定各车间由哪个班组主导，或由两个班组协作。

第二阶段：在确定分配后，对每个班组分别调度，再合并结果。

#### 方法二：启发式策略

优先让距离目标车间更近的班组承担该车间主要工序。

根据距离表：

| 车间 | 较近班组 |
|---|---|
| A | 班组1 |
| B | 班组2 |
| C | 班组1 |
| D | 班组2 |
| E | 班组1 |

#### 方法三：整数规划模型

引入 0-1 变量表示工序之间的先后顺序，通过析取图（Disjunctive Graph）构造精确调度模型，并利用分支定界法求解。

## 9. 问题4建模思路：追加预算购买设备并进一步优化

### 9.1 问题特征

问题4在问题3基础上增加设备采购决策。

已知条件：

- 追加预算为 500000 元；
- 各类设备单价已知；
- 需要决定给班组1和班组2各增购多少台、什么类型的设备；
- 采购后重新调度，使完成全部任务时间最短。

### 9.2 决策变量

设：

```math
\Delta N_k^{(g)}
```

表示给班组 \(g\) 增购 \(k\) 类设备的台数。

要求：

```math
\Delta N_k^{(g)}\in\mathbb{Z}_{\ge0}
```

### 9.3 预算约束

```math
\sum_{g\in\{1,2\}}\sum_{k\in K}\Delta N_k^{(g)}\cdot price_k\le 500000
```

### 9.4 更新设备数量

```math
\hat{N}_k^{(g)}=N_k^{(g)}+\Delta N_k^{(g)}
```

### 9.5 目标函数

```math
\min Makespan(\hat{N}_k^{(g)})
```

问题4因此可视为一个双层优化问题：

| 层级 | 任务 |
|---|---|
| 外层 | 决定采购方案 |
| 内层 | 在采购后的设备配置下求最优调度时长 |

### 9.6 求解策略

#### 策略一：瓶颈分析

先从问题3的最优或较优调度结果中识别瓶颈设备，即关键路径上最紧张的设备类型。

优先考虑增购瓶颈设备。

#### 策略二：边际收益分析

对每类设备，计算增加 1 台设备带来的 Makespan 减少量。

边际收益可定义为：

```math
MR_k=\frac{\Delta Makespan_k}{price_k}
```

选择边际收益更高的设备类型优先购买。

#### 策略三：枚举法

由于预算有限，最便宜设备单价为 35000 元/台，最多可购买约 14 台，因此可以枚举所有满足预算约束的采购方案。

对每种采购方案：

1. 更新班组1和班组2的设备数量；
2. 调用问题3的调度算法；
3. 计算 Makespan；
4. 选择 Makespan 最小的方案。

### 9.7 背包模型表达

设预算为 \(B=500000\)，设备类型 \(k\) 的单价为 \(c_k\)，采购数量为 \(d_k\)，则：

```math
\sum_k c_k d_k \le B,\quad d_k\in\mathbb{Z}_{\ge0}
```

对每种合法采购组合，调用调度算法，选择完成时间最短的组合。

## 10. 关键技术点汇总

### 10.1 时间精确计算

所有工序持续时间计算必须：

1. 先用小时制计算：

```math
工序时长=\frac{工程量}{台数\times效率}
```

2. 转换为秒：乘以 3600；
3. 使用 ceil 函数向上取整。

统一表达为：

```math
T=\left\lceil \frac{Q}{n\cdot v}\times3600\right\rceil
```

### 10.2 并行工序时间取最大值

双设备工序中，两类设备各自独立完成工程量，工序完成时间取两者最大值：

```math
T_{op}=\max(T_X,T_Y)
```

### 10.3 转运时间准确计算

转运时间为：

```math
T_{trans}=\left\lceil \frac{d}{2}\right\rceil
```

如果设备在时间 \(F_{L1}\) 完成位置 \(L_1\) 的任务，到达 \(L_2\) 的时间为：

```math
t_{arrive}=F_{L1}+T_{trans}(L_1,L_2)
```

该设备参与 \(L_2\) 工序的开始时间不能早于 \(t_{arrive}\)。

### 10.4 资源约束处理

调度时需要维护每台设备的状态：

| 状态 | 含义 |
|---|---|
| 最早可用时间 | 设备下一次可以参与作业的最早时间 |
| 当前所在位置 | 用于计算下一次转运时间 |
| 设备类型 | 用于匹配工序需求 |
| 所属班组 | 用于初始出发距离和问题3/4输出 |

设备分配时，优先选择满足类型要求且最早可用时间较小的设备。

### 10.5 C车间循环处理

C3-C5 重复3遍时，必须拆为 9 道独立工序，并添加链式约束：

```text
C3_1 → C4_1 → C5_1 → C3_2 → C4_2 → C5_2 → C3_3 → C4_3 → C5_3
```

对应约束包括：

```math
S_{C3,2}\ge F_{C5,1}
```

```math
S_{C3,3}\ge F_{C5,2}
```

## 11. 推荐方法与适用场景

| 方法 | 适用场景 | 优点 | 局限 |
|---|---|---|---|
| 枚举法 | 问题1，设备台数少 | 精确，计算量小 | 不适合大规模多车间调度 |
| 关键路径法 CPM | 问题2/3，构建调度优先级 | 能识别关键工序链 | 属于启发式，不保证全局最优 |
| 列表调度算法 | 问题2/3，资源分配 | 实现简单，易于输出调度表 | 结果可能只是近似最优 |
| 整数规划 ILP | 精确求解 | 理论严谨，结果精确 | 变量多，计算量大 |
| 贪心 + 局部搜索 | 问题4采购决策 | 快速、可落地 | 可能陷入局部最优 |
| 模拟退火 / 遗传算法 | 大规模复杂问题 | 具备全局搜索能力 | 参数调节复杂，稳定性依赖实现 |

## 13. 答题与实现注意事项

1. 每个问题应单独建模，变量、约束和目标函数不要混用。
2. 对双设备工序，必须明确两类设备各自独立完成工程量，工序时间取最大值。
3. 所有时间计算必须精确到秒，并向上取整。
4. 跨车间转运时间和设备初始出发时间必须纳入调度。
5. C车间 C3-C5 循环必须拆解为 9 道独立工序处理。
6. 问题4的采购方案必须分班组列出购买台数，并验证总费用不超过 500000 元。
7. 结果填表时，需要确保每台设备的起始时间、结束时间、持续时间和工序编号一致。
8. 持续工作时间应对应设备实际作业时间，不应混入设备转运时间。

## 15. 小结

该建模思路的核心是将 B 题转化为多资源受限调度问题。问题1可以用枚举法求解；问题2和问题3需要处理多车间、多设备、跨车间运输与资源冲突，可采用关键路径优先的列表调度；问题4进一步加入设备采购决策，可通过瓶颈分析、边际收益分析和枚举采购方案进行优化。

整个模型的关键在于四点：

1. 双设备工序完成时间取最大值；
2. 同车间工序按顺序执行；
3. 跨车间转运时间必须计入设备到达时间；
4. C3-C5 循环必须拆解成独立工序链。
