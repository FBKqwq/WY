# Q1 开发文档

## 1. 问题目标

- 由**班组 1**独立承担** A 车间**全部整修任务。
- 建立调度模型，计算完成 A 车间所有工序的**最短时长（Makespan，秒）**。
- 在表 1 中输出每台设备执行各工序的：**序号、设备编号、起始时间、结束时间、持续工作时间(s)、工序编号**（见 `outputs/表1_问题1调度结果.xlsx`）。

依据：`src/Qbase/competitionB.md` 第六节「问题一逐句解析」及 B 题通用假设。

## 4. 核心实现逻辑

- 仅保留 `所属车间 == A` 的原始工序，按工序顺序与重复规则展开为 `ExpandedOperation` 列表。
- 设备池为**班组 1**全部设备；初始位置为 `班组1`，与距离表节点一致。
- **单设备工序**：枚举该类型所有设备，取 `结束时间` 最早者（同分按字典序稳定）。
- **双设备工序**：枚举类型 1、类型 2 的设备有序对（排除同设备编号），使 `max(结束1, 结束2)` 最小；两类设备各自完成全工程量，工序完成时间为两者最大结束时间；**不**人为插入双机之间的额外等待。
- 同车间连续作业转运时间为 0；自 `班组1` 或跨车间移动使用 `ceil(距离/速度)` 秒。

## 5. 重复工序处理

- 单行 `重复执行次数` > 1：展开为 `工序编号#1…#R`。
- 同一车间内多行**相同**重复次数且 `工序顺序` 连续整数：按「轮次 × 工序」交错展开（如 C3→C4→C5 各 3 次）。

## 6. 双设备工序处理

- 每条需求生成独立设备任务记录；作业时长分别按各自效率与全工程量 `ceil(Q/v×3600)`（单位 m³/h）计算。

## 7. 运输时间处理

- 使用 `车间距离表.xlsx` 三列格式（起点、终点、距离）构建对称矩阵。
- 转运时间 `calc_transport_time = ceil(d / v)`，与持续作业时间分列记录。

## 8. 已实现约束

- [x] 车间内工序顺序（按展开后的 `sequence_key`）
- [x] 重复工序展开（含多块轮询）
- [x] 设备类型匹配
- [x] 双设备共同完成（取 max 完工）
- [x] 设备互斥（同一设备时间线不重叠）
- [x] 跨节点转运时间（班组↔车间、车间↔车间；同车间为 0）
