# Q4 开发文档

## 1. 问题目标

- 在新增设备预算不超过 `500000` 元约束下，同时确定设备购买方案与作业调度策略。
- 使用班组1和班组2设备完成 A、B、C、D、E 五个车间全部整修任务，最小化总完工时长（Makespan）。
- 输出：
  - 表4：调度明细（序号、设备编号、起始时间、结束时间、持续工作时间(s)、工序编号、班组）
  - 表5：购买方案（设备名称、班组1购买台数、班组2购买台数、购买总费用）

依据：`src/Qbase/competitionB.md` 第九节“问题四逐句解析”。

## 4. 核心算法

### 4.1 调度求解

- 使用贪心事件推进调度：
  - 同车间内严格按工序顺序执行；
  - 单设备工序选择完成时间最早设备；
  - 双设备工序枚举设备组合，工序完成时间取两条设备任务结束时间最大值；
  - 同车间运输时间为0，跨车间运输时间按 `ceil(distance / speed)`。

### 4.2 购买方案搜索

- **有界完全枚举**：在「增购总台数 ≤ 上界」且总价不超过预算的前提下，DFS 遍历所有非负整数购买向量，叶节点调用问题4调度，取 `(makespan, 总费用)` 字典序最小为枚举层最优。
- **束搜索**：以「不购买」与「枚举层最优」为初始束，每层对每个状态尝试各 `(班组, 设备类型)` 加购 1 台并重调度，按 `(makespan, cost)` 保留宽度内的若干互异方案，捕捉枚举上界之外的加购路径。
- **多起点贪心**：按利用率排序的瓶颈类型多种轮换顺序，每步在全部可购键上加 1 台选最优，与束搜索互补。
- **爬山加购**：在合并后的最优方案上，在预算内反复尝试任意键 +1 台直至单步无法缩短工期。

说明：在当前数据与同一套贪心调度规则下，边际上「多买几台」未必继续缩短 makespan（瓶颈被调度方式饱和时会出现）；搜索仍会在预算内尽量压低工期，若最优解只用到部分预算，表5 中购买总费用会低于 50 万上限，这是目标（最小化工期）优先于「花光预算」的结果。

## 5. 已处理约束

- [x] 工序顺序固定（按编号顺序，含重复展开后顺序）
- [x] 重复工序展开（含“重复3次”语义）
- [x] 双设备共同完成（两设备均完成后工序完成）
- [x] 设备类型匹配
- [x] 设备互斥
- [x] 跨车间运输时间约束
- [x] 持续工作时间(s)不含运输时间
- [x] 预算约束（购买总费用不超过500000）

## 8. 已知问题

- 调度内核为贪心事件推进，购买方案搜索在固定该内核下枚举/束搜索/爬山组合，不保证联合全局最优；若更换调度器，最优购买组合可能变化。
- 若需进一步压缩工期，可考虑 CP-SAT 等精确模型与调度联合建模（复杂度显著上升）。
