# Q1 开发文档

## 1. 问题目标

- 由**班组 1**独立承担** A 车间**全部整修任务。
- 建立调度模型，计算完成 A 车间所有工序的**最短时长（Makespan，秒）**。
- 在表 1 中输出每台设备执行各工序的：**序号、设备编号、起始时间、结束时间、持续工作时间(s)、工序编号**（见 `outputs/表1_问题1调度结果.xlsx`）。

依据：`src/Qbase/competitionB.md` 第六节「问题一逐句解析」及 B 题通用假设。

## 4. 核心实现逻辑

- 仅保留 `所属车间 == A` 的原始工序，按工序顺序与重复规则展开为 `ExpandedOperation` 列表。
- 设备池为**班组 1**全部设备；初始位置为 `班组1`，与距离表节点一致。
- **单设备工序**：枚举投入台数 \(n=1,\ldots,N\)（\(N\) 为该类型在班组中的台数）。对选定 \(n\)，取「到达车间门限」最小的 \(n\) 台设备，公共开工时刻为 `max(车间就绪, 第 n 小的 可用+转运)`；持续作业时间按 `calc_work_duration(Q, n×单台效率, 单位)`，即**同类并联**分摊工程量。在表 1 中为所选的每台设备各写一行。
- **双设备工序**：枚举 \((n_1,n_2)\)，持续时间为各类 `calc_work_duration(Q, n_k×效率_k, 单位)`；开工时刻为两类门限的公共下界；工序 Makespan 贡献为 `max(dur1,dur2)`。每台参与设备一条记录；若两类时长不同，各类设备的 `结束时间` 可不同（与赛题假设(2)及表1 示例一致）。**不**人为插入两类设备之间的额外等待。
- 同车间连续作业转运时间为 0；自 `班组1` 或跨车间移动使用 `ceil(距离/速度)` 秒。

## 5. 重复工序处理

- 单行 `重复执行次数` > 1：展开为 `工序编号#1…#R`。
- 同一车间内多行**相同**重复次数且 `工序顺序` 连续整数：按「轮次 × 工序」交错展开（如 C3→C4→C5 各 3 次）。

## 6. 双设备工序处理

- 每条需求生成独立设备任务记录；作业时长分别按各自效率与全工程量 `ceil(Q/v×3600)`（单位 m³/h）计算。

## 7. 运输时间处理

- 使用 `车间距离表.xlsx` 三列格式（起点、终点、距离）构建对称矩阵。
- 转运时间 `calc_transport_time = ceil(d / v)`，与持续作业时间分列记录。

## 12. 最优性与适用范围（原「已知问题」）

**问题 1（当前数据与建模口径下）：Makespan 为全局最优最短工期。**

论证要点（与第 4 节实现一一对应）：

1. **工序顺序固定**：展开后的 A 车间工序链顺序由赛题与数据表确定，不存在「调换工序先后」的决策分支。
2. **每道工序的决策空间有限且已穷尽**：单设备类枚举 \(n\in\{1,\ldots,N\}\)；双设备类枚举 \((n_1,n_2)\)。对每组台数，作业时长由 `ceil` 规则与工程量、效率唯一确定。
3. **在固定台数下开工时刻最优**：同类并联需等待所选的每一台都「可用 + 转运」到位；在 \(n\) 台同型设备中，使 \(\max_i(\mathrm{avail}_i+\mathrm{trans}_i)\) 最小的选法必为：按门限升序取前 \(n\) 台（实现中的 `_gates_to_workshop` 排序后取前缀），公共开工为 \(\max(\text{车间就绪},\text{第 }n\text{ 小的门限})\)。
4. **无跨工序耦合损失**：A1 用到的灌装机/输送臂在官方 A 车间后续工序中不再被复用；A2、A3 同理按类型独占使用。故「逐步对每道工序选全局最小完工」与「整条链 Makespan 最小」一致，无需前瞻或 CP-SAT 也能得到 Q1 的全局最优。

**不适用上述结论的情形（须在文档或代码中另行说明）：**

- 附件变更导致 A 车间出现**更多工序**、**同类型设备在多道工序间复用**且存在资源竞争时，仅对单工序枚举未必等于整条链最优，应改用语义搜索 / CP-SAT 或专用分解算法。
- **问题 2 / 问题 3** 若仍调用同一 `solve_problem1` 框架做多车间循环，该框架一般为**可行解启发式**，**不**在此文档承诺全局最优（见对应 `Q2/DEV.md`、`Q3/DEV.md`）。

**可选增强（非 Q1 必需）：** 若需对变更后的数据自动做最优性检查，可增加「枚举状态数 / 与下界对比」的断言或单元测试；在复杂场景下再接入 OR-Tools CP-SAT（见技能说明中的求解策略分级）。
